## ğŸ“‹ File Type Explanations

### .godot Files

#### `project.godot`
- Main configuration file
- Defines app name, main scene, settings
- Input mappings
- Display settings
- Similar to Unity's ProjectSettings

### .tscn Files (Scene Files)
- Text-based scene format
- Contains node hierarchy
- Node properties
- Connected signals
- Can be version controlled easily
- Similar to Unity's .prefab/.unity files

### .gd Files (GDScript)
- Python-like scripting language
- Typed or dynamic (your choice)
- Built-in to Godot
- Fast and optimized
- Similar to Unity's C# scripts

### .import Files
- Auto-generated by Godot
- Contains import settings for assets
- Don't edit manually
- Should be in version control

## ğŸ¯ File Organization Best Practices

### Scenes (`scenes/`)
**Organize by type:**
- `ui/` - All UI-related scenes
- `gameplay/` - Core gameplay scenes
- `characters/` - Player & NPC scenes
- `levels/` - Level scenes
- `effects/` - Particle effects, VFX

**Naming convention:**
- `snake_case.tscn`
- Descriptive names: `main_menu.tscn`, not `menu1.tscn`

### Scripts (`scripts/`)
**Organize by system:**
- Match scene organization
- Group related scripts together
- One script per file
- Name matches functionality

**Naming convention:**
- `snake_case.gd`
- Descriptive: `player_controller.gd`, not `pc.gd`

### Assets (`assets/`)
**Organize by type:**
- `sprites/` - Images
- `audio/` - Sound files
- `fonts/` - Typography
- `shaders/` - Custom shaders
- `themes/` - UI themes

**Naming convention:**
- `snake_case` or `kebab-case`
- Include size for sprites: `player_64x64.png`
- Include purpose: `button_hover.png`

## ğŸ’¡ Tips

1. **Keep scenes small** - Break complex scenes into subscenes
2. **Use scene inheritance** - Powerful Godot feature
3. **Organize early** - Easier to maintain structure from start
4. **Use groups** - Tag nodes for easy finding
5. **Resource files** - Use .tres for data (like ScriptableObjects)
6. **Consistent naming** - Pick a convention and stick to it
